1. The evaluation of a good site

The good site: Digi.no

Checklist 1:

  • Page title is OK.
  • Image text alternatives ("alt text") is OK.
  • Text:
    • Headings are used OK. Only h2 is used.
    • Contrast ratio is OK.
    • Resize Text is OK. Functionality of the browser.
  • Interaction:
    • Keyboard access and visual focus is OK.
    • "Forms, labels, and errors" are OK.
  • General:
    • The site does not contain multimedia alternatives. OK
    • Basic Structure Check is OK.

Checklist 2 (pdf):

  • Authority and Accuracy
    Digi.no is the author of the page. The url says that the page is norwegian(.no). The qualifications of the group that created the site are that they are journalists and it professionals.
  • Purpose and Content
    The purpose of the site is to inform people about the new and interesting things that happen in the IT frontiers. The main purpose is to be a newspaper, informational. Contact information is readily available.
  • Currency
    The website is regularly updated. Probably last friday. The currency of this site is it's lifeline. The site is well maintained. Links are not broken.
  • Design, Organization, and Ease of Use
    The site seems well organized and functional overall.

2. The evaluation of a poor site

The poor site: Trondheim Mikrobryggeri

Checklist 1:

  • Page titles are OK
  • Image alt text is non-existing, except for the sites Facebook button.
  • Text:
    • Some headings are in use, but there are text that should be headings which are instead resized with the font-tag.
    • Contrast and readability is bad since text is placed directly over an image so the gray text on a white part of the image is very difficult to read.
    • Resizing the text is OK.
  • Interaction:
    • It is not possible to tab to all parts of the page, but you can tab away from the ones you can tab to. The tab order is not logical, there is no visual focus and one cannot tab into dropdown lists.
    • Form keyboard access is OK, and so is the form labels. Error handling was not checked since the only forms on the site would either send an email, or post to the site's guestbook so we chose not to.
  • General:
    • There is no multimedia on the site, so no need to check for multimedia alternatives.
    • The basic structure check is mostly OK. The text makes sense, but headings are often missing.

Checklist 2 (pdf):

  • Authority and Accuracy
    The author of this website is Mikrobryggeriet AS, as stated in the copyright notice on the bottom of the page. Could not find any information about the qualifications of the creators.
  • Purpose and Content
    This is a company or organization web site. The purpose of the site is to present the pub/restaurant Mikrobryggeriet AS, aka Trondheim Mikrobryggeri, and what they have to offer. Information about opening hours and how to contact them is provided. The content is balanced, objective and factual. Contact information is provided.
  • Currency
    Not possible to tell when the web site was last updated. The currency of the site is not very important to the type of information on the site. Opening hours are listed for the current week, so they are always up to date. Would be nice to know how current the menu is, but that's impossible to tell.
  • Design, Organization and Ease of Use
    The web site does not seem to be well maintained, no links are broken, but there is some missing content, e.g. the food menu. The appearence overall is that the site is poorly designed, difficult to read, help screens, search functions and site map are all unavailable.

3. Project proposal

Primary Goal

We purpose to create a site for beer brewing. Sort of a personal page for us to post our progress and tips as well as a resource for people interested in brewing their own beer.

Pages of the project:

  • Home
  • About
  • Equipment and Setup
  • HowTo Brew: a brewing tutorial
  • Basic Ingredients
    • Yeast
    • Grains
    • Hops
  • A brief history of beer
  • Recipes
    • Basic
    • IPA
    • Dry Sout
    • ... more to come
  • Resources, useful links for further information

Audience

Fellow brewers and people that has not brewed before. Old browsers are not a major concern as we mostly focus on basic technology and well tested design. With basic frameworks we don't need to focus on device optimization.

Multimedia content

Some YouTube videos and other images will be used. But this will be added as embedded html. So we don't need to think too much about the multimedia content.

Interactions and Dynamic features

Interesting features we think about are alcohol percentage calculators, and other simple calculations to setup your beer. If we have really good time we can add a recipes builder, so you can create your beer online before brewing it.

Content

As brewing is a hobby of ours so we create most of the content ourselves. Information about different grain and hops types might be taken from Wikipedia or other sources.

What to learn from different sites

Different sites like this are focused mostly in america so we should have a more national focus. And probably have content in norwegian.
